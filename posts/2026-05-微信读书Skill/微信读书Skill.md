# 让 AI 读懂你的阅读：微信读书 Skill

我平时用微信读书看书，看到好的段落就划线，偶尔写几句想法。时间一长，划线和想法攒了不少，但真正回去翻看的时候很少——微信读书的笔记界面不算好用，想按章节整理出来更是费劲。

后来我发现了一个东西叫 Skill，说白了就是给 AI 装上的新能力。有了微信读书的 Skill，AI 能直接读到我微信读书里的划线、想法、书架、阅读统计，甚至能帮我把笔记按章节导出成 Markdown 文件。

这篇文章聊聊 Skill 是什么，微信读书 Skill 能做什么，以及我日常怎么用它。

## Skill 是什么

Skill 这个词翻译成中文是「技能」，在 AI 的语境里，它指的是一种扩展 AI 能力的方式。

大模型本身能聊天、写文章、分析问题，但它不知道你微信读书里有什么书，不知道你划了哪些线，不知道你的日历上有什么安排。Skill 就是解决这个问题的——它给 AI 接上了新的数据源和工具。

你可以把 Skill 理解成手机上的 App。手机出厂时有基本功能，打电话、发短信，但你想点外卖得装外卖 App，想听歌得装音乐 App。AI 也一样，出厂时能聊天，但你让它查微信读书，它得先装上对应的 Skill。

Skill 的安装很简单。我在用的是一个叫 QClaw 的 AI 助手，它支持通过命令行安装 Skill。比如安装微信读书的 Skill，就是一条指令的事。装完之后，AI 就拥有了对应的能力，你在对话中自然地提需求就行，不需要手动调接口、看文档。

Skill 背后的技术原理不复杂。大多数 Skill 做的事情是：把某个服务的 API 封装起来，告诉 AI 什么时候该调用、怎么调用、拿到结果怎么展示。对用户来说，这些细节完全透明。你只需要说「帮我看看我在微信读书的笔记」，AI 就知道该调哪个接口、传什么参数、怎么把结果整理好给你看。

有一点值得说：Skill 不是插件市场的概念。它更像是给 AI 的能力清单上加上一项。你装了一个 Skill，AI 的知识边界就扩大了一点。你可以装很多个，它们之间不冲突，甚至可以配合使用——比如你同时装了微信读书 Skill 和笔记导出 Skill，AI 就能在读取划线之后直接帮你整理成文件。

## 微信读书 Skill

微信读书 Skill 目前有两个：一个是「微信读书助手」，负责搜索、书架、笔记、书评、阅读统计这些；另一个是「微信读书笔记导出」，专门把划线和想法按章节整理成 Markdown 文件。

### 安装

安装需要两步：装 Skill 本身，配置 API Key。

Skill 的安装是一行命令。在 QClaw 里，告诉 AI「安装微信读书 Skill」就行，它会自动处理。

API Key 是微信读书提供的认证凭证，绑定你的微信读书账号。拿到 Key 之后设置一个环境变量：

```
export WEREAD_API_KEY=wrk-xxxxxxxx
```

配置好之后，AI 就能以你的身份访问微信读书的数据了。Key 是你自己的，别人拿不到。

### 微信读书助手

这个 Skill 的能力覆盖了微信读书的大部分功能：

**搜索书籍。** 在微信读书的书城里搜索，支持模糊匹配。比如你说「帮我搜一下三体」，它会返回匹配的书名、作者、评分，你确认之后再做后续操作。

**查看书架。** 看你书架上有多少本书，有哪些正在读、哪些读完了。微信读书的书架不区分书和有声书，Skill 也会把有声书统计进来。

**阅读统计。** 这是我觉得最有意思的功能之一。可以查某个月读了多久书、今年读了几本、阅读偏好是什么类型。微信读书年度报告里那些数据，现在随时都能问 AI 要。

**笔记和划线。** 能查看你在某本书里有多少划线和想法，也能看每章的划线内容。这个功能是笔记导出 Skill 的基础。

**热门划线和书评。** 能看其他读者在某本书、某个章节的热门划线，也能看公开的书评和点评。不过我个人的阅读习惯是不看这些——我喜欢带着空白去读，不想被别人的划线影响第一印象。

**推荐。** 基于你的阅读历史做个性化推荐，或者找跟某本书相似的书。

所有这些能力，在对话中直接用自然语言触发就行。比如「我这个月读了多久」「帮我看看三体的笔记」「给我推荐几本类似的书」，AI 会自动判断该调哪个接口。

### 笔记导出

这个 Skill 做的事情很专注：输入一个书名，自动把这本书的划线和想法按章节整理成 Markdown 文件。

导出的文件长这样：

```
# 书名

作者：xxx
导出时间：2026-05-20

---

## 第一章标题

> 划线内容1

*—— 2026-05-18*

> 划线内容2

*—— 2026-05-19*

> 划线原文

**想法：** 这段话让我想到……

*—— 2026-05-19*
```

有几个细节值得一提：

- 想法会附带对应的划线原文，不会单独悬空
- 没有笔记的章节自动跳过，不生成空白章节
- 文件末尾有统计：多少个章节有笔记，多少条划线，多少条想法
- 如果搜索到的版本没有笔记，会自动从书架匹配有笔记的版本

导出一条命令就搞定：

```
python3 export_notes.py "绝对坦率"
```

## 我怎么用微信读书 Skill

说实话，装上 Skill 的第一天我只是试了试，觉得「哦挺方便的」，但没觉得有多大用。真正觉得值，是开始写读后感之后。

### 写读后感时的笔记回看

我写读后感的习惯是先把书的笔记过一遍，回忆自己读的时候划了哪些线、想了哪些事。以前这个过程是在微信读书 App 里翻，划线列表很长，想找到特定章节的内容得来回滚。现在直接让 AI 导出笔记，按章节排好的 Markdown 文件，在电脑上打开一目了然。

比如写《面纱》的读后感时，我让 AI 导出了笔记。毛姆这本书我划了三十多条线，散在十几个章节里。导出之后按章节看，很快就能回忆起我当时对哪个情节有感触、哪段话让我停下来想了想。比在 App 里翻效率高很多。

而且 Markdown 文件可以搜。想找某个关键词相关的划线，直接全文搜索就行，比微信读书 App 里的搜索好用。

### 阅读统计的自我审视

微信读书的年度报告每年底才出一次，平时想知道自己的阅读数据得自己算。有了 Skill 之后，随时可以问 AI：这个月读了多久？今年读了几本？最近在读什么类型的书？

我偶尔会问一下，不是为了打卡，就是想确认一下自己的阅读节奏还在不在。有时候忙起来一两周没看书，问一下统计就会意识到「该找本书读了」。算是一种温和的自我提醒。

### 「给我推荐几本」

这个需求以前是打开微信读书翻推荐页，或者在豆瓣上看书单。现在直接跟 AI 说「给我推荐几本书」，它会基于我的阅读历史推荐。推荐的精准度我不确定比微信读书自带的推荐好多少，但好处是可以在对话中继续追问——「有没有类似《战国歧途》这种讲历史但写得不枯燥的？」这种需求，推荐页做不到。

还有一个小用法：读完一本好书之后，趁热问一句「跟这本书类似的还有什么」，往往能发现一些自己不会搜到的书。这种「读完了还想读类似」的感觉，以前得靠豆瓣的相关推荐，现在在对话里就解决了。

### 不怎么用的功能

热门划线和书评我基本不看。前面说过，我读书不喜欢被别人的划线影响。微信读书的阅读界面里会显示热门划线，我都是关掉的。Skill 有这个能力，但我选择不用——能力在那里，用不用是我的事。

书架查看也用得少。我知道自己书架上有啥，真要查的时候再问。

## 小结

Skill 的核心价值就一点：让 AI 能访问到你的数据。微信读书 Skill 让 AI 能看到你读了什么、划了什么、想了什么。一旦数据流通了，AI 就不再只是一个聊天工具，而是一个真正了解你阅读习惯的助手。

笔记导出是我目前最常用的功能。不是为了导出而导出，是因为写读后感需要回看笔记，而 Markdown 格式的笔记比微信读书 App 里的笔记界面好用太多。工具的价值从来不在功能本身，在于它能不能让你本来要做的事变得更顺畅。

回头想，Skill 这件事的意义不只是「多了几个功能」。以前微信读书的数据被锁在 App 里，我只能在 App 的框架下使用它们。Skill 打开了这扇门，数据变成了我可以自由处理的格式，AI 也能基于这些数据给我更具体的帮助。从「App 里有什么用什么」到「我的数据我想怎么用就怎么用」，这个转变比单个功能本身更值得关注。

如果你也在用微信读书，也写读后感，可以试试。
