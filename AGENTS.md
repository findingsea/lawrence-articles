# Lawrence 的文章库 (lawrence-articles)

## 项目说明

本文档库用于存放书籍读后感与评价类文章，辅助 AI 生成符合 Lawrence 写作风格的草稿。

## 目录结构

```
lawrence-articles/
├── draft/     # 草稿（AI 生成，待用户微调）
├── post/      # 正式发布的文章（用户确认后移动至此）
└── image/     # 图片资源
```

## 文件命名

草稿：draft/YYYY-MM-书名.md
正式：post/YYYY-MM-书名.md

## 写作流程

1. 用户输入：提供对某本书的看法、观点、思考片段（可以是零散的）
2. AI 搜索：搜索书籍信息 + 豆瓣评价 + 书评
3. AI 生成：整合用户观点与搜索内容，生成初稿
4. 用户微调：用户基于草稿修改，形成最终版本
5. 正式发布：用户将文件移动到 post/ 目录

## 文章结构规范

每篇文章应包含：

1. 前言
   - 交代读这本书的契机或最初的感触
   - 可以引用在朋友圈/豆瓣发过的短评

2. 正文（分点展开）
   - 每个小节围绕一个核心观点展开
   - 结合书的内容，但不能止于情节复述
   - 重点分析写法、形式、技巧（如叙事手法、人物塑造、矛盾设置等）
   - 善用对比和关联（关联其他书籍、电影、作品）

3. 结语
   - 总结性评价或推荐语

## 行文风格（根据历史文章总结）

- 语气：真诚、有深度、个人视角鲜明
- 第一人称：大量使用"我觉得"、"我想尝试讲的是"
- 不人云亦云：有自己的独特观点，不止于复述情节
- 善用对比：经常关联其他作品（金庸、《冰与火之歌》、《阿甘正传》等）
- 干净利落：无 emoji，纯文字
- 篇幅：初稿 800-1500 字，可根据内容调整
- 用词：中文为主，英文术语自然融入

## 禁止事项

- 不做流水账式的情节复述
- 不堆砌摘抄原文
- 不写空洞的"这本书很好，推荐大家读"类结语
