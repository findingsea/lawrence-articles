# Lawrence 的文章库 (lawrence-articles)

## 项目说明

本文档库用于存放书籍读后感与评价类文章，辅助 AI 生成符合 Lawrence 写作风格的草稿。

## 目录结构

```
lawrence-articles/
├── AGENTS.md          # 本文件：项目规范与写作指南
├── draft/             # 草稿（AI 生成，待用户微调）
│   ├── YYYY-MM-文章主题.md          # 单篇文章草稿
│   └── YYYY-MM-文章主题/            # 复杂文章子目录
│       ├── memo.md                   # 讨论记录、素材汇总
│       └── 文章主题.md               # 正式草稿（与文件夹同名）
├── post/              # 正式发布的文章（用户确认后移动至此）
│   └── YYYY-MM-文章主题.md
└── image/             # 图片资源
```

## 命名规范

### 文件夹命名
- **格式**：`YYYY-MM-文章主题/`
- **示例**：`2026-04-ai-era-engineer-adaptation/`
- **规则**：
  - 年份-月份开头（4位年份 + 2位月份）
  - 使用英文小写，单词用连字符分隔
  - 简洁明了，能概括文章核心内容

### 文件命名

| 文件类型 | 命名格式 | 示例 |
|---------|---------|------|
| 单篇草稿 | `draft/YYYY-MM-文章主题.md` | `draft/2026-04-backend-engineer-ai-era.md` |
| 子目录草稿 | `draft/YYYY-MM-主题/文章主题.md` | `draft/2026-04-ai-era-engineer-adaptation/ai-era-engineer-adaptation.md` |
| memo 文件 | `draft/YYYY-MM-主题/memo.md` | `draft/2026-04-ai-era-engineer-adaptation/memo.md` |
| 正式文章 | `post/YYYY-MM-文章主题.md` | `post/2026-04-ai-era-engineer-adaptation.md` |

**注意**：
- 子目录中的草稿文件**必须与文件夹同名**（去掉日期前缀）
- memo 文件**保持原名** `memo.md`，用于记录讨论和素材

## 协作写作流程

```
讨论 → memo → 草稿 → 用户修改 → 再讨论 → 正式文章
         ↑___________________________________↓
```

1. **讨论阶段**：用户与 AI 就文章主题进行讨论
2. **记录到 memo**：AI 将讨论内容整理到 `draft/YYYY-MM-主题/memo.md`
3. **生成草稿**：AI 基于 memo 生成 `draft/YYYY-MM-主题/主题.md`
4. **用户修改**：用户基于草稿进行修改，添加个人观点
5. **迭代讨论**：继续讨论完善
6. **正式发布**：定稿后移动到 `post/YYYY-MM-主题.md`

## memo 文件用途

`draft/YYYY-MM-主题/memo.md` 是**讨论记录和素材汇总**，包含：
- 行业趋势调研
- 核心问题讨论（Q&A 形式）
- 参考资料链接
- 灵感/想法记录
- 待办事项

**注意**：memo 是工作文件，不是最终文章。正式文章基于 memo 生成。

## 文章结构规范

### 深度书评（单本书）

1. **前言**
   - 交代读这本书的契机或最初的感触
   - 可以引用在朋友圈/豆瓣发过的短评
   - 或直接点出这本书最打动你的点

2. **正文（分点展开）**
   - 每个小节围绕一个核心观点展开
   - 结合书的内容，但不能止于情节复述
   - 重点分析**写法、形式、技巧**（叙事手法、人物塑造、矛盾设置、结构编排等）
   - 善用对比和关联（关联其他书籍、电影、作品）
   - 可以引用原文让作者观点自己说话

3. **结语**
   - 总结性评价或推荐语
   - 不要空洞，要有具体指向

### 月度阅读报告（多本书）

- 每本书：豆瓣链接 + 封面图 + 简短评价（200-500字）
- 评价精准，一针见血
- 不面面俱到，只讲最突出的点

### 技术/观点类文章

- **前言**：交代背景和写作动机
- **正文**：分点论述，逻辑清晰
- **结语**：总结观点或给出建议

## 行文风格（根据历史文章总结）

### 核心原则

**1. 个人视角鲜明**
- 大量使用"我觉得"、"我认为"、"我想尝试讲的是"
- 不回避个人感受和偏好
- 敢于表达负面评价（"推荐度比较低"、"有点令人失望"）

**2. 不剧透，注重分析（书评适用）**
- 明确说："故事内容本身不用多讲，一来是剧透了再读的乐趣就少得多"
- 重点分析**写法、形式、技巧**（叙事手法、人物塑造、矛盾设置）
- 跳脱内容猎奇，关注"作者怎么把故事讲得令人手不释卷"

**3. 善用对比和关联**
- 通过关联其他作品/案例突出主题
- 阅读量大，信手拈来
- 常见参照：金庸、《冰与火之歌》、《阿甘正传》、《哈利波特》等

**4. 有深度但不高冷**
- 用通俗语言讲清楚观点
- 不故弄玄虚，不掉书袋
- "写的非常浅显易懂，即便是文科生看懂整个逻辑也是没有任何问题"

**5. 犀利直接**
- 敢说真话，不人云亦云
- "舒尔茨明显不是一个好作家"
- "刘瑜到底也没带来新的惊喜"
- "口味彻底被养叼了"

### 语言特征

- **干净利落**：无 emoji，纯文字
- **中文为主**：英文术语自然融入
- **篇幅灵活**：短评几百字，深度文章可达上万字
- **善用引用**：引用原文/数据让观点更有说服力
- **豆瓣链接（书评）**：书名必带豆瓣链接，阅读报告配封面图

## 禁止事项

- 不做流水账式的情节复述
- 不堆砌摘抄原文
- 不写空洞的"这本书很好，推荐大家读"类结语
- 不故弄玄虚、掉书袋
- 不人云亦云，没有自己的观点
