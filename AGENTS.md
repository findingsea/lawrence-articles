# Lawrence 的文章库 (lawrence-articles)

## 项目说明

本文档库用于存放书籍读后感与评价类文章，辅助 AI 生成符合 Lawrence 写作风格的草稿。

## 目录结构

```
lawrence-articles/
├── draft/     # 草稿（AI 生成，待用户微调）
├── post/      # 正式发布的文章（用户确认后移动至此）
└── image/     # 图片资源
```

## 文件命名

草稿：draft/YYYY-MM-书名.md
正式：post/YYYY-MM-书名.md

## 写作流程

1. 用户输入：提供对某本书的看法、观点、思考片段（可以是零散的）
2. AI 搜索：搜索书籍信息 + 豆瓣评价 + 书评
3. AI 生成：整合用户观点与搜索内容，生成初稿
4. 用户微调：用户基于草稿修改，形成最终版本
5. 正式发布：用户将文件移动到 post/ 目录

## 文章结构规范

### 深度书评（单本书）

1. **前言**
   - 交代读这本书的契机或最初的感触
   - 可以引用在朋友圈/豆瓣发过的短评
   - 或直接点出这本书最打动你的点

2. **正文（分点展开）**
   - 每个小节围绕一个核心观点展开
   - 结合书的内容，但不能止于情节复述
   - 重点分析**写法、形式、技巧**（叙事手法、人物塑造、矛盾设置、结构编排等）
   - 善用对比和关联（关联其他书籍、电影、作品）
   - 可以引用原文让作者观点自己说话

3. **结语**
   - 总结性评价或推荐语
   - 不要空洞，要有具体指向

### 月度阅读报告（多本书）

- 每本书：豆瓣链接 + 封面图 + 简短评价（200-500字）
- 评价精准，一针见血
- 不面面俱到，只讲最突出的点

## 行文风格（根据历史文章总结）

### 核心原则

**1. 个人视角鲜明**
- 大量使用"我觉得"、"我认为"、"我想尝试讲的是"
- 不回避个人感受和偏好
- 敢于表达负面评价（"推荐度比较低"、"有点令人失望"）

**2. 不剧透，注重分析**
- 明确说："故事内容本身不用多讲，一来是剧透了再读的乐趣就少得多"
- 重点分析**写法、形式、技巧**（叙事手法、人物塑造、矛盾设置）
- 跳脱内容猎奇，关注"作者怎么把故事讲得令人手不释卷"

**3. 善用对比和关联**
- 通过关联其他作品突出本书特点
- 阅读量大，信手拈来
- 常见参照：金庸、《冰与火之歌》、《阿甘正传》、《哈利波特》等

**4. 有深度但不高冷**
- 用通俗语言讲清楚观点
- 不故弄玄虚，不掉书袋
- "写的非常浅显易懂，即便是文科生看懂整个逻辑也是没有任何问题"

**5. 犀利直接**
- 敢说真话，不人云亦云
- "舒尔茨明显不是一个好作家"
- "刘瑜到底也没带来新的惊喜"
- "口味彻底被养叼了"

### 语言特征

- **干净利落**：无 emoji，纯文字
- **中文为主**：英文术语自然融入
- **篇幅灵活**：短评几百字，深度书评可达上万字
- **善用引用**：引用原文让作者观点自己说话
- **豆瓣链接**：书名必带豆瓣链接，阅读报告配封面图

---

## 禁止事项

- 不做流水账式的情节复述
- 不堆砌摘抄原文
- 不写空洞的"这本书很好，推荐大家读"类结语
- 不故弄玄虚、掉书袋
- 不人云亦云，没有自己的观点

## 参考范例

### 深度书评
- 《巨人的陨落》：分析史实与虚构、多线叙事、节奏与厚度、矛盾与冲突
- 《异类》：机遇、文化传承两大主题分章节展开
- 《纳什均衡与博弈论》：批评性书评，指出内容与期待的差距

### 月度阅读报告
- 一月份阅读报告：4本书，每本精准点评
- 四月份阅读报告：9本书，优缺点明确
